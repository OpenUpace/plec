## 1. How to contribute?
This is a good question. We have two ways now: 
- 1. Core code: You can clone plec to your computer, compile it, find bugs, and fix them. Another way is: 
- 2. Document review: While coding, we often write documentation incorrectly (including this). So we need document reviewers to review the docs.
You might say: "But I don't know what my first step is." Okay, you can find a 'good first issue' tag in issue to contribute. Don't be nervous. 

## 2. What is the format for issues or pull requests? 
Similar to Angular git commits, we also have this format: 

[Tag] Main message. 

First, `[Tag]` is about the message, e.g. Question, PR, etc. 

Second, use markdown for code blocks.

Finally, welcome to contributing!
